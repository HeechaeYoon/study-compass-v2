# START HERE — Codex One-Shot Build Package

## 목적

이 패키지는 중학교 2학년 자기주도학습 수업용 웹앱을 **기존 저품질 구현을 부분 수정하지 않고, 시각 설계와 기능 구조를 처음부터 다시 세워 완성**하도록 Codex에 제공하는 실행 문서 묶음이다.

최우선 목표는 다음 두 가지를 동시에 만족하는 것이다.

1. `reference/design-board.png`의 따뜻한 종이 질감, 카드 구성, 손그림 장식, 학습 노트 분위기를 시각적으로 매우 가깝게 재현한다.
2. 16문항 진단, 5축 점수, 대표 성향, 상세 리포트, AI 프롬프트, 복사·저장·삭제를 실제로 안정적으로 동작시킨다.

이 문서 묶음은 “첫 결과를 내고 멈추는 구현”을 허용하지 않는다. Codex는 계획, 구현, 이미지 생성, 브라우저 확인, 시각 비교, 기능 테스트, 자체 리뷰를 반복하고 `docs/QA_ACCEPTANCE.md`의 종료 조건을 통과한 뒤에만 완료를 선언해야 한다.

## 패키지 배치

이 폴더의 전체 내용을 대상 Git 저장소 루트에 복사한다.

```text
repo-root/
  AGENTS.md
  PRD.md
  PLAN.md
  START_HERE.md
  docs/
  reference/
  .prompts/
```

기존 프로젝트가 있다면 먼저 Git 상태를 확인한다. 기존 구현은 낮은 완성도의 프로토타입으로 간주하며, CSS를 덧붙여 억지로 보정하지 않는다. 안전한 백업 또는 Git 이력을 확보한 뒤 UI 구조를 새로 만든다.

## Codex 실행 전 필수 조건

다음 기능을 사용할 수 있는 환경을 권장한다.

- 이미지 입력: 참조 스크린샷을 분석하기 위해 필요
- `$imagegen`: 종이 질감과 연필 장식 자산 생성에 필요
- 브라우저 또는 in-app browser: 실제 렌더링 확인에 필요
- Playwright: 화면 캡처와 시각 회귀 검증에 필요
- Subagents: 시각, 기능, 접근성 검토를 분리하기 위해 권장

`$imagegen`을 사용할 수 없다면 전체 화면을 임시 그림으로 대체하지 않는다. CSS와 SVG로 구현 가능한 범위는 진행할 수 있지만, 연필·질감 자산 품질을 충족하지 못할 가능성이 있으면 코드 작업을 시작하기 전에 그 제약을 명확히 보고한다.

## Codex에 입력할 단일 시작 프롬프트

아래 프롬프트와 이 패키지 전체를 한 번에 제공한다.

```text
이 저장소의 기존 웹앱은 부분 개선하지 말고, 제공된 설계 문서를 기준으로 처음부터 다시 설계하고 구현해줘.

반드시 먼저 AGENTS.md와 START_HERE.md를 읽고, 문서에 지정된 순서로 전체 스펙을 읽어.
reference/design-board.png를 핵심 시각 기준으로 사용해.
$imagegen을 명시적으로 사용해서 필요한 장식 자산을 생성하되, 전체 UI를 이미지로 만들지 말고 종이 질감과 연필 같은 독립 자산만 생성해.
생성 프롬프트와 선택 이유를 .prompts에 기록해.

이 작업은 첫 구현으로 끝내지 마.
PLAN.md를 업데이트하면서 다음을 모두 스스로 수행해:
1. 기존 코드와 Git 상태 점검
2. 새 UI 아키텍처와 데이터 구조 구현
3. 4개 핵심 화면 구현
4. 문항·점수·유형·프롬프트·저장 기능 구현
5. Playwright로 1280x800 기준 화면 캡처
6. 참조 이미지와 비교하고 시각적으로 반복 개선
7. Vitest, typecheck, lint, build 실행
8. 시각·기능·접근성 관점의 subagent 리뷰
9. 리뷰 지적사항 수정
10. docs/QA_ACCEPTANCE.md의 종료 조건을 모두 통과한 뒤 최종 보고서 작성

중간에 단순한 첫 버전을 보여주고 멈추지 마.
명세에 없는 사소한 결정은 docs/DECISION_LOG.md의 원칙에 따라 스스로 결정해.
사용자에게 질문하는 것은 작업을 진행할 수 없는 실제 차단 요인이 있을 때만 허용해.
완료 시 실행 방법, 테스트 결과, 시각 검증 결과, 생성 자산 목록, 남은 한계를 FINAL_REPORT.md에 정리해.
```

같은 내용은 `.prompts/INITIAL_CODEX_PROMPT.md`에도 들어 있다.

## 성공의 의미

다음 상태가 아니면 완료가 아니다.

- 참조 이미지의 분위기만 흉내 낸 일반적인 SaaS UI가 아니다.
- 기본 컴포넌트 라이브러리 티가 나지 않는다.
- 모든 화면이 같은 디자인 언어를 공유한다.
- 1280×800에서 레이아웃, 여백, 카드 비율, 컬러, 장식 밀도가 참조와 매우 가깝다.
- 16문항 응답부터 결과 저장·삭제까지 실제로 동작한다.
- 개인 응답과 메모가 외부로 전송되지 않는다.
- 자동 테스트와 브라우저 검증이 통과한다.
- Codex가 자체 리뷰 후 발견한 문제를 수정한 상태다.

## 정직한 한계

원본 Figma, 폰트 파일, 벡터 자산이 없으므로 브라우저 렌더링을 PNG와 픽셀 단위로 100% 동일하게 만드는 것은 보장할 수 없다. 이 패키지는 그 한계를 감추지 않고 다음 방식으로 재현 확률을 높인다.

- 기준 viewport 고정
- 세부 디자인 토큰 명시
- 화면별 좌표와 크기 명세
- 고립된 자산만 ImageGen으로 생성
- SVG와 CSS 중심의 재사용 가능한 UI 구현
- Playwright 캡처와 시각 차이 반복 검토
- 별도 시각 리뷰 에이전트와 종료 기준 적용
