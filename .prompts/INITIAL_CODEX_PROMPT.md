# Initial Codex Prompt

이 저장소의 기존 웹앱은 부분 개선하지 말고, 제공된 설계 문서를 기준으로 처음부터 다시 설계하고 구현해줘.

반드시 먼저 `AGENTS.md`와 `START_HERE.md`를 읽고, 문서에 지정된 순서로 전체 스펙을 읽어.
`reference/design-board.png`를 핵심 시각 기준으로 사용해.

`$imagegen`을 명시적으로 사용해서 필요한 장식 자산을 생성하되, 전체 UI를 이미지로 만들지 말고 종이 질감과 연필 같은 독립 자산만 생성해. 생성 프롬프트와 선택 이유를 `.prompts/IMAGEGEN_ASSETS.md`에 기록해.

이 작업은 첫 구현으로 끝내지 마. `PLAN.md`를 업데이트하면서 다음을 모두 스스로 수행해.

1. 기존 코드와 Git 상태 점검
2. 새 UI 아키텍처와 데이터 구조 구현
3. 4개 핵심 화면의 정적 시각 셸 구현
4. 문항·점수·유형·프롬프트·저장 기능 구현
5. Playwright로 1280×800 기준 화면 캡처
6. 참조 이미지와 비교하고 반복 개선
7. Vitest, typecheck, lint, build 실행
8. 시각·기능·접근성 관점의 subagent 리뷰
9. 리뷰 지적사항 수정
10. `docs/QA_ACCEPTANCE.md`의 종료 조건을 모두 통과한 뒤 최종 보고서 작성

중간에 단순한 첫 버전을 보여주고 멈추지 마.
명세에 없는 사소한 결정은 `docs/DECISION_LOG.md`의 원칙에 따라 스스로 결정해.
사용자에게 질문하는 것은 작업을 진행할 수 없는 실제 차단 요인이 있을 때만 허용해.

완료 시 실행 방법, 테스트 결과, 시각 검증 결과, 생성 자산 목록, 남은 한계를 `FINAL_REPORT.md`에 정리해.
