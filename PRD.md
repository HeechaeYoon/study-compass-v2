# PRD — 자기주도학습 성향 탐색 웹앱

- 문서 상태: Implementation-ready
- 제품 단계: v1 static web app
- 대상: 중학교 2학년
- 주요 사용 기기: 학교 태블릿 가로 화면, 노트북·PC
- 배포: GitHub Pages
- 핵심 참조: `reference/design-board.png`

---

## 1. 제품 요약

학생이 16개의 객관식 문항에 답하면 현재 학습 습관과 전략 사용 경향을 5개 축으로 정리하고, 가장 가까운 대표 학습 성향과 성장 포인트를 보여준다. 결과를 바탕으로 학생은 다음 수업에서 Gemini 같은 AI 챗봇에 붙여넣을 자기설명 프롬프트를 생성한다.

이 앱은 학생을 분류하는 검사가 아니다. 결과는 다음 행동을 정하기 위한 코칭 출발점이다.

핵심 질문:

- 나는 어떤 방식으로 공부할 때 잘 되는 편인가?
- 어떤 부분을 보완하면 자기주도학습이 더 쉬워질까?
- AI에게 내 상황을 어떻게 설명해야 더 구체적인 학습 전략을 받을 수 있을까?

---

## 2. 배경과 문제

수업에서 “자기주도학습”을 설명만 하면 학생은 자신의 실제 행동과 연결하기 어렵다. 반대로 가벼운 유형 테스트만 제공하면 결과가 흥미 요소에 머물고 다음 학습 행동으로 이어지지 않는다.

이 제품은 다음 간극을 해결한다.

1. 추상적인 자기주도학습 개념을 학생의 일상 행동 언어로 바꾼다.
2. 결과를 고정 유형이 아니라 현재 전략 사용 패턴으로 해석한다.
3. 강점과 성장 포인트를 짧은 실천 미션으로 연결한다.
4. 학생이 AI 챗봇에게 자신의 학습 조건을 설명할 수 있도록 구조화된 프롬프트를 제공한다.
5. 로그인과 서버 없이 학교 환경에서 빠르게 접속하고 사용할 수 있게 한다.
6. 수업용 접속 코드와 Daisy 저작권 표시로 링크의 무단 공유·복제·재배포를 완화한다.

---

## 3. 제품 목표

### G-01. 자기이해

학생이 자신의 현재 학습 전략을 5개 축으로 이해한다.

### G-02. 성장 행동

학생이 가장 중요한 성장 포인트 한 가지와 즉시 실행할 미션을 얻는다.

### G-03. 다음 수업 연결

학생이 과목, 단원, 목표를 입력해 AI 챗봇용 프롬프트를 복사할 수 있다.

### G-04. 안전한 수업 도구

학생의 응답과 메모가 외부 서버로 전송되지 않는다.

### G-05. 높은 시각 완성도

참조 이미지의 따뜻한 학습 노트·종이·손그림 분위기를 유지해 학생의 참여를 유도하되, 유치하거나 게임처럼 보이지 않는다.

## 5. 사용자

### Primary user

중학교 2학년 학생

특성:

- 개인 태블릿 또는 학교 제공 태블릿 사용
- 긴 설문보다 한 문항씩 진행하는 인터페이스에 적합
- 성격 유형처럼 단정되는 표현에 민감할 수 있음
- 학습전략 전문 용어보다 행동 중심 문장이 필요
- 복사·붙여넣기와 AI 챗봇 사용 경험이 있을 가능성이 높음

### Secondary user

출강 강사 또는 교사

필요:

- 설치 없이 링크 제공
- 수십 명 동시 접근
- 짧은 시간 안에 완료
- 결과를 다음 AI 수업으로 연결
- 개인정보 이슈를 최소화
- 학교망·브라우저 제약에 대응

## 6. 사용 환경

- Padlet 또는 Canva 수업 자료의 링크로 진입
- 최초 접속과 정적 파일 다운로드에 인터넷 사용
- 앱 로직은 클라이언트에서 실행
- 기준 화면: 1280×800
- 1차 지원 범위: 1024×768 이상 가로형 화면
- 보조 지원: CSS viewport 560–899px 스마트폰 가로 화면은 compact landscape 모드로 사용 가능
- 보조 지원: CSS viewport 360px 이상 스마트폰 세로 화면은 반응형 세로 모드로 사용 가능
- 권장 브라우저: 최신 Chrome, Edge, Safari
- CSS viewport 360px 미만 세로 화면 또는 560px 미만 가로 화면에서는 더 넓은 화면 또는 가로 모드 사용 안내 표시

```text
링크 접속
→ 수업 접속 코드 입력
→ 앱 목적·개인정보 안내 확인

→ 16문항 응답
→ 결과 요약 확인

→ 필요 시 결과 이미지 또는 로컬 저장
→ 공유 기기라면 저장 결과 삭제
```

## 8. 정보 구조

### 공통 앱 헤더

- 앱 제목: `나의 공부 스타일을 탐색하는 시간`
- 설명: `자기주도학습 유형 진단`
- 우측 메타 정보:
  - `총 16문항`
  - `5가지 학습 축`
  - `약 5~7분 소요`
- 손그림 별 장식과 노란 밑줄

### 화면 S1. 시작

- 목적 설명
- 닉네임 선택 입력
- 시작 버튼
- 5축 소개 카드
- 개인정보 보호 요약
- 자세한 안내 접기/펼치기
- 저장 결과가 있을 때 다시 보기·삭제

### 화면 S2. 문항

- 진행 문항 수
- 진행률
- 예상 남은 시간
- 축 태그
- 문항
- 4개 또는 5개 선택 카드
- 이전·다음
- 정직한 응답 안내

### 화면 S3. 결과 요약

- 완료 메시지
- 대표 학습 성향
- 현재 응답 기준 안내
- 핵심 설명
- 레이더 차트
- 5축 이름·수준
- 성장 포인트
- 강점·균형·추천 전략 카드
- 상세 영역 진입

### 화면 S4. AI 프롬프트

- 4가지 프롬프트 모드: `공부 계획`, `개념 학습`, `계획 이미지`, `개념 이미지`
- 목표, 상황, 어려움, 원하는 도움 입력
- 과목·단원·이번 학습 목표 입력 지원
- 학생 메모 자동 반영
- 프롬프트 미리보기
- 복사 버튼
- 결과 저장·이미지 저장·삭제

## 9. 기능 요구사항

### FR-001 시작 및 안내

- 시작 화면 전에 수업 접속 코드 입력 화면을 표시한다.
- 유효한 접속 코드는 로컬 브라우저에 만료 시각과 함께 저장할 수 있다.
- 접속 코드는 정적 앱 안에서 검증되는 수업용 억제책이며, 서버 기반 인증을 대체하지 않는다.

- 이름 또는 별명은 선택 입력이다.
- 빈 값으로도 시작할 수 있다.
- 입력값은 서버로 보내지 않는다.
- 개인정보 안내는 짧은 요약과 상세 접기 영역으로 제공한다.
- 저장 결과가 있으면 시작 화면에서 접근할 수 있다.

### FR-002 문항 진행

- 총 16문항
- 한 화면에 한 문항
- 12개 동의형, 4개 상황형
- 동의형은 4점 척도
- 상황형은 5개 선택지 가능
- 중립 동의 선택지는 제공하지 않는다.
- 역문항은 사용하지 않는다.
- 응답하지 않으면 다음 버튼을 비활성화한다.
- 이전으로 돌아가도 응답을 유지한다.
- 문항 이동 시 스크롤 위치를 상단으로 맞춘다.
- 브라우저 뒤로가기로 응답이 소실되지 않게 앱 상태를 관리한다.

### FR-003 점수

- 5축 원점수 산출
- 질문 정의로부터 축별 최소·최대 자동 계산
- 0~100 정규화
- 학생 화면에서는 숫자를 중심 정보로 사용하지 않는다.
- 숫자가 필요한 차트에서도 우열감을 줄이는 표현을 사용한다.
- 라벨:
  - 70~100: 강점
  - 40~69: 균형
  - 0~39: 성장 포인트

### FR-004 대표 성향

- 8개 기준 프로필과 학생 벡터를 비교한다.
- 최고 축 하나만으로 정하지 않는다.
- 특수 규칙 후 유클리드 거리 기반 매칭을 적용한다.
- 대표 성향은 한 개 표시한다.
- 1위와 2위가 가까우면 보조 성향을 설명에 반영한다.
- 항상 `현재 답변 기준으로`라는 표현을 사용한다.

### FR-005 결과 요약

첫 화면에 다음을 표시한다.

- 대표 학습 성향
- 짧은 핵심 설명
- 5축 시각화
- 강점 요약
- 주의할 점
- 추천 학습법
- 가장 중요한 성장 포인트 1개

### FR-006 상세 리포트

접을 수 있는 상세 영역에 다음을 표시한다.

- 대표 성향과 보조 성향
- 5축별 해석
- 주요 강점
- 주의할 점
- 잘 맞는 학습법
- 피하면 좋은 학습 방식
- 이번 성장 포인트
- 다음 수업 성장 미션
- 결과가 고정된 특성이 아니라는 안내

### FR-007 학생 메모

- 제목: `내가 보기엔 다른 점`
- textarea 제공
- 개인 기록으로 취급
- 메모가 입력되면 AI 프롬프트에 자동 반영
- 이름, 연락처, 민감한 개인정보를 쓰지 않도록 안내
- 저장 시 메모를 함께 저장
- 삭제 시 함께 삭제

### FR-008 AI 프롬프트

입력 필드:

- 과목
- 단원
- 이번 학습 목표
- 현재 상황
- 어려운 점
- 원하는 도움
- 내가 보기엔 다른 점

요구:

- 일부 또는 전체 입력이 없어도 기본 프롬프트 생성
- 대표 성향, 5축 라벨, 강점, 성장 포인트, 주의점, 추천 방식 포함
- 16개 문항의 질문과 선택 답변 전체 포함
- 30~40분 실행계획 요청
- 개념 설명·예시·오개념·확인문제 중심 학습용 프롬프트 생성
- Gemini 또는 ChatGPT 이미지 생성 기능에 붙여넣을 인포그래픽 프롬프트 생성
- 자기점검 질문 3개 요청
- 후속 질문 예시 2개 요청
- plain text로 복사 가능
- 복사 실패 fallback 제공

### FR-009 결과 저장

- 명시적 저장 버튼을 눌렀을 때만 localStorage에 저장
- 저장 날짜 표시
- 저장 결과 다시 보기
- 현재 결과 삭제
- 저장 결과 전체 삭제
- 삭제 전 확인 modal
- 삭제 후 복구 불가 안내
- 여러 결과를 지원할 필요는 없으며 v1은 최신 1개 저장으로 단순화 가능

### FR-010 결과 이미지

- 결과 요약 카드만 PNG 내보내기
- 학생 메모와 자유 입력은 기본 이미지에서 제외
- 브라우저 제한으로 실패할 수 있으므로 보조 기능
- 실패 시 복사 기능 안내
- 외부 폰트·이미지 CORS 문제 없이 로컬 자산만 사용

### FR-011 오류·복구

- localStorage 접근 실패 시 앱은 계속 동작
- Clipboard API 실패 시 수동 선택 fallback
- 이미지 저장 실패 시 기능 전체가 중단되지 않음
- 저장된 데이터 버전이 맞지 않으면 안전하게 무시하고 삭제 옵션 제공
- 새로고침 전 응답 복구는 필수가 아니지만, 같은 탭 내 상태는 유지

### FR-012 저작권 표시와 관리자 코드 생성

- 모든 화면에 `© Daisy Teacher. All rights reserved. 무단 복제 및 재배포 금지` 문구를 학생 흐름을 방해하지 않는 수준으로 표시한다.
- 결과 이미지와 상세 리포트 복사본에도 같은 문구를 포함한다.
- AI 프롬프트 본문에는 저작권 문구를 붙이지 않는다.
- 숨겨진 관리자 진입 동작으로 마스터코드 입력 모달을 열 수 있다.
- 마스터코드 원문은 코드에 직접 노출하지 않고 빌드 환경변수 `MASTER_CODE`에서 파생된 검증값만 앱에 포함한다.
- 관리자 모드에서는 1일부터 90일까지 일 단위 유효 기간을 지정해 수업 접속 코드를 생성한다.
- 이 기능은 정적 GitHub Pages 앱의 수업용 억제책이며, 전문적인 접근 제어가 필요한 경우 백엔드 검증이 필요하다.

## 10. 비기능 요구사항

### NFR-001 개인정보

- 외부 네트워크 전송 없음
- 분석도구 없음
- 추적 cookie 없음
- nickname, answers, result, memo를 URL에 넣지 않음
- 브라우저 저장 위치와 삭제 방법 명시

### NFR-002 성능

- 정적 파일 기반
- 초기 JS 번들은 가능한 작게 유지
- 장식 이미지는 압축된 WebP/PNG 사용
- 최초 화면에서 불필요한 라이브러리 지연 로딩
- 1280×800 기준 조작 시 체감 지연 없음

### NFR-003 접근성

- 키보드 탐색
- 명확한 focus ring
- 색상 외 텍스트 라벨
- 충분한 대비
- form label 연결
- `aria-live`로 복사·저장 결과 안내
- reduced-motion 대응
- 터치 타깃 최소 44px 권장

### NFR-004 브라우저

- 최신 Chromium
- 최신 Safari/WebKit
- 최신 Edge
- GitHub Pages subpath 지원

### NFR-005 시각 품질

- 일반 템플릿 UI처럼 보이지 않음
- 참조 이미지와 동일한 디자인 언어
- 4개 화면 간 스타일 일관성
- 핵심 화면별 시각 리뷰 점수 90/100 이상
- 어떤 시각 평가 항목도 최소 기준 미만이면 완료 불가

## 11. 콘텐츠 원칙

- 학생 친화적이지만 유치하지 않은 존댓말
- 명령보다 제안 표현
- `가능성이 있어요`, `해보면 좋아요` 사용
- `너는 원래`, `반드시`, `부족하다`, `낮다`, `열등하다` 금지
- 성향명은 차분한 명사형
- 가벼운 비유는 설명에만 사용
- 결과마다 실제 행동 1개 이상 포함

## 12. 성공 기준

### 제품 성공

- 학생이 7분 내외에 진단을 완료할 수 있다.
- 결과 화면에서 대표 성향과 성장 포인트를 구분해 이해할 수 있다.
- AI 프롬프트를 별도 편집 없이 복사할 수 있다.
- 저장하지 않은 데이터는 앱 종료 후 서버 어디에도 남지 않는다.

### 구현 성공

- 모든 자동화 검사 통과
- GitHub Pages 빌드·배포 가능
- 1280×800에서 참조와 매우 유사
- 1024×768과 1440×900에서 큰 레이아웃 붕괴 없음
- 시각·기능·접근성 리뷰 지적사항 수정 완료

## 13. 연구·해석 한계

- 이 도구는 검증된 심리검사가 아니다.
- 짧은 자기보고 응답은 실제 행동과 다를 수 있다.
- 과목, 단원, 최근 경험, 컨디션에 따라 결과가 달라질 수 있다.
- 기준 프로필과 임계값은 수업용 초기 설계값이다.
- 실제 학생 파일럿 후 문항 이해도, 결과 분포, 유형 집중 현상을 검토해야 한다.
- AI 챗봇의 추천은 정답이 아니며 학생이 현실 조건에 맞게 조정해야 한다.
